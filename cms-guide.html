General Advice & FAQs

How should I use sections in this CMS?

This CMS is based on reusable code sections. To create a new page, copy the required section code and paste it into your new page. You can update the content (text, images, icons) as needed.

Can I rearrange sections?

Yes, you can place sections in any order based on your page requirements. Sections can be reused and repositioned freely.

Can I modify design elements like colors, fonts, or button styles?

No, design elements such as button colors, typography, spacing, and layout styles are fixed to maintain consistency across the website. Any design-related changes require code-level updates.

What should I do if I need design changes?

If you need to modify styles (e.g., button color, layout, spacing), please contact the developer responsible for this project. These changes cannot be done through the CMS.

Can I reuse sections across multiple pages?

Yes, all sections are reusable. You can copy the same section and use it on different pages without any issue.

How should I update content?

Only update the content inside the section (text, images, links). Avoid removing or modifying the core structure of the code, as it may break the layout or functionality.

Is this a dynamic CMS with URLs?

No, this setup does not use dynamic CMS URLs. All pages must be created manually by reusing section code and updating the content accordingly.

What should I do if I face issues?

If you face any issues while editing or creating pages, please reach out to the developer responsible for this CMS guide.

CMS User Notes

How can I add or update content in a CMS section?

To create or update a page using a CMS section, follow the steps below:

  • Identify the section you want to use from the section list.
  • Create a new page and copy the structure/code of the required section.
  • Paste it into your new page and update the content, including headings, text, images, and icons, as per your requirements.
  • Ensure all content aligns with your page goal and design consistency.

If you encounter any issues while updating or implementing this section, please reach out to the developer responsible for creating this CMS guide.
